# RULE_PRIORITY_MATRIX

## Objectif

Séparer les conflits entre règles FACTORY.

---

## HARD_BLOCKERS

- POOL_COLLISION
- DUPLICATES
- CHECK_FACTORY_FORMAT
- QA_STATUS=FAIL

Action :
- blocage immédiat

---

## SOFT_WARNINGS

- CHECK_THEME_BALANCE
- CHECK_FLOW
- CHECK_REPEAT_DENSITY

 Action :
- warning
- révision possible

---

## OPTIONAL_OPTIMIZERS

- rythme parfait
- variété maximale
- optimisations micro

 Action :
- non bloquant
